# 7-Day User Training Curriculum

Audience: new operators using the clone daily.

## Day 1 - Orientation and Baseline

Goal: understand the interface and establish a baseline workflow.

Exercises:
1. Log in and visit Dashboard, Talk, Integrations, Tasks.
2. Run 3 core Talk prompts (priority, approvals, 2-hour plan).
3. Record baseline metrics:
   - Open tasks
   - Pending approvals
   - Unread inbox

Exit criteria:
- User can navigate all core pages without guidance.
- User can produce one usable daily plan from Talk mode.

## Day 2 - Task and Approval Discipline

Goal: create reliable execution loops.

Exercises:
1. Clean and re-prioritize task list.
2. Process pending approvals with rationale.
3. Use prompt: `Convert these priorities into executable tasks with order and due dates.`

Exit criteria:
- Clear top 3 priorities are visible.
- Approval queue is current.

## Day 3 - Inbox and Communication Control

Goal: use the clone for decision-safe communication.

Exercises:
1. Sync inbox.
2. Draft replies for urgent threads.
3. Validate each draft before approval.

Exit criteria:
- User can produce concise, context-correct drafts.
- No unreviewed auto-send behavior.

## Day 4 - Memory and Personalization

Goal: improve output quality by training persistent context.

Exercises:
1. Add 5 high-value memory items:
   - Communication tone
   - Decision constraints
   - Priority themes
   - Team preferences
   - Risk boundaries
2. Remove stale memory.

Exit criteria:
- Clone responses reflect user's tone and constraints.

## Day 5 - Integrations and Data Trust

Goal: connect only needed systems and verify data flow.

Exercises:
1. Connect required integrations.
2. Run one sync path and verify records in dashboard views.
3. Investigate one failed/empty sync and resolve cause.

Exit criteria:
- At least one critical integration path is healthy.
- User can identify sync-health signals.

## Day 6 - Advanced Prompting and Decision Quality

Goal: move from generic prompts to operator-grade prompts.

Exercises:
1. Use structured prompts:
   - outcome
   - constraints
   - time horizon
   - output format
2. Compare weak vs strong prompts and note quality delta.

Exit criteria:
- User consistently requests ranked, actionable outputs.

## Day 7 - Weekly Review and Operating Standard

Goal: lock in repeatable weekly habits.

Exercises:
1. Run weekly clone training endpoints.
2. Review scores, summary, and dispatch recommendations.
3. Write a personal operating standard (daily + weekly cadence).

Exit criteria:
- User has a documented cadence and review loop.
- User can run weekly training flow end-to-end.

## Coach Checklist (for admins)

- Observe prompt quality (specificity, constraints, expected format).
- Track turnaround on approvals and priority completion.
- Confirm memory hygiene once per week.
- Ensure user can recover from integration/sync issues.

## Certification Rubric (Pass/Needs Work)

Pass if user can:
1. Produce daily top-3 priorities with rationale.
2. Maintain approvals without backlog growth.
3. Generate safe email drafts and review before action.
4. Keep memory updated and improve response quality.
5. Run weekly training and interpret outcomes.
